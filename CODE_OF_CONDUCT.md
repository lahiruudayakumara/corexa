# Corexa Code of Conduct

## Our Commitment

Corexa is committed to building a respectful, inclusive, and sustainable open source community.

We want contributors, maintainers, and users to feel safe participating regardless of experience level, identity, background, or professional status.

## Expected Behavior

- be respectful and constructive
- assume good intent while being honest about problems
- welcome questions from new contributors
- give actionable feedback instead of dismissive criticism
- keep technical disagreement focused on ideas and outcomes
- help maintain a collaborative and developer-first environment

## Unacceptable Behavior

- harassment or intimidation
- personal attacks
- discriminatory or exclusionary language
- bad-faith arguments or deliberate disruption
- publishing private information without permission
- hostile conduct in issues, pull requests, discussions, chat, or review threads

## Scope

This Code of Conduct applies across:

- GitHub issues
- pull requests
- discussions
- review comments
- project documentation
- community or maintainer communication spaces connected to Corexa

## Reporting

If you experience or witness behavior that violates this Code of Conduct, contact the maintainers privately at the channel defined in [SUPPORT.md](SUPPORT.md) or the security contact flow in [SECURITY.md](SECURITY.md) if confidentiality is important.

## Enforcement

Maintainers may take any action they deem appropriate, including:

- warning a contributor
- editing or removing harmful content
- temporarily restricting participation
- permanently banning a contributor from project spaces

Enforcement decisions will prioritize community safety and project health.

## Maintainer Standards

Maintainers are expected to model the behavior described here, especially when giving technical feedback, managing conflict, or closing discussions.
