# Corexa Roadmap

This roadmap translates the project vision into staged product and platform delivery.

## Phase 1: Local Platform Foundation

Goals:

- ship the desktop chat shell
- establish Corexa Runtime inference integration
- implement repository scanning and indexing
- add semantic search and local memory foundations
- establish CLI and runtime API baselines

Exit criteria:

- runtime health and chat flows are stable
- contributors can navigate the monorepo and run the baseline toolchain
- repository intelligence can progress from discovery to persisted index artifacts

## Phase 2: Autonomous Engineering Workflows

Goals:

- add multi-agent orchestration
- support autonomous coding workflows
- ship reviewer and debugger loops
- introduce terminal approvals and replay
- stabilize plugin contracts

Exit criteria:

- agent runs can plan, execute, review, and recover from failures
- terminal activity is observable and policy-aware
- extension points are coherent enough for internal and early external use

## Phase 3: Platform and Enterprise Expansion

Goals:

- add enterprise policy systems
- build plugin ecosystem infrastructure
- support hybrid local and remote routing
- introduce cloud sync and team memory concepts
- deepen observability and fleet operations

Exit criteria:

- governance and runtime controls can support larger organizations
- plugins and integrations can be built against stable contracts
- Corexa can operate as both a local product and a broader platform
