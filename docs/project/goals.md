# Corexa Goals

## Mission

Build the best local-first AI-native development platform for serious software engineering.

## Strategic Goals

### 1. Local-first execution

Corexa should allow developers to work with AI capabilities without depending on continuous cloud connectivity.

### 2. Repository-native intelligence

Corexa should understand real codebases through parsing, indexing, memory, and dependency context rather than relying on generic chat alone.

### 3. Trusted autonomy

Corexa should support autonomous engineering workflows while keeping humans in control through approvals, auditability, and strong workspace boundaries.

### 4. Extensible platform surface

Corexa should support desktop, CLI, SDK, plugins, and future enterprise integrations from a shared architectural core.

### 5. Open source credibility

Corexa should feel like a real long-term open source platform with contributor-friendly docs, maintainable structure, and transparent project direction.

## Product Principles

- local first
- modular by default
- observable by design
- secure by default
- repo aware, not prompt naive
- autonomy with guardrails
- premium developer experience

## Near-term Success Criteria

- local runtime can serve assistant and agent workflows
- repository scans are incremental and reliable
- desktop and CLI surfaces share contracts instead of diverging
- contributors can navigate the repo without tribal knowledge
- docs explain both the vision and the actual build sequence

## Non-goals For The Early Phase

- being a general-purpose consumer chat app
- supporting every model backend on day one
- building a marketplace before core workflows are reliable
- optimizing for growth hacks over architectural durability
