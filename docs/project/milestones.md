# Corexa Milestones

## Milestone 0: Open Source Foundation

Objective:

Establish Corexa as a contributor-ready public repository with strong project documentation and baseline automation.

Success signals:

- repository has contributor, governance, support, and security docs
- issue and PR intake is structured
- Dependabot and CI are configured
- docs provide a clear map of project direction

## Milestone 1: Local Runtime Alpha

Objective:

Deliver a functioning local runtime surface with health, model, workspace, and streaming chat APIs.

Success signals:

- runtime daemon starts reliably
- desktop and CLI can query runtime state
- core model and chat contracts are stable

## Milestone 2: Repository Intelligence Alpha

Objective:

Give Corexa structured awareness of real repositories.

Success signals:

- file discovery and ignore rules work predictably
- syntax-aware chunks and symbols are indexed
- semantic retrieval returns relevant repository context

## Milestone 3: Desktop Workflow Beta

Objective:

Ship a usable AI-native desktop shell for repository-centered development workflows.

Success signals:

- desktop boot and dev flow are stable
- workspace panels, chat, and model selection are coherent
- terminal sessions and runtime diagnostics are visible

## Milestone 4: Agent Workflow Beta

Objective:

Introduce multi-agent engineering flows with review and debugging loops.

Success signals:

- role-based orchestration executes predictably
- reviewer and debugger loops catch real failures
- approval checkpoints work across terminal and file changes

## Milestone 5: Plugin and Enterprise Readiness

Objective:

Turn Corexa from an app into an extensible platform.

Success signals:

- plugin contracts are stable enough for external development
- governance and security controls support broader adoption
- hybrid runtime and team-oriented workflows are feasible
