# Support

## Where To Go

Use the right channel so maintainers and contributors can help effectively.

### Bug Reports

Use GitHub Issues with the bug report template.

### Feature Requests

Use GitHub Issues with the feature request template.

### Architecture Discussions

Use GitHub Discussions for design questions, direction proposals, or ecosystem brainstorming.

### Documentation Improvements

Use the documentation issue template or open a pull request directly for small fixes.

### Security Issues

Use the private disclosure flow in [SECURITY.md](SECURITY.md).

## What Support Includes

The project can usually help with:

- repository setup questions
- development workflow questions
- architecture clarification
- reproducible bugs
- roadmap direction

## What Support Does Not Guarantee

- custom feature delivery timelines
- private debugging for unsupported environments
- immediate triage for every open-ended product idea

## Good Support Requests

A strong request usually includes:

- clear problem description
- expected versus actual behavior
- relevant environment details
- logs or screenshots if applicable
- links to affected docs, issues, or PRs

## Maintainer Expectations

Maintainers will try to keep communication respectful, clear, and actionable, but response times may vary depending on project bandwidth and severity.
